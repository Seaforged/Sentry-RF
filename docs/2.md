Then implement operational modes: STANDARD, COVERT, and HIGH_ALERT.
Mode definitions:

STANDARD: current behavior — full CAD + RSSI sweep + WiFi scanner + OLED + buzzer + LED. This is the default boot mode.
COVERT: RF scanning only. WiFi scanner OFF (call esp_wifi_stop() and esp_wifi_deinit() to fully deinitialize — zero RF emissions from WiFi). OLED OFF (or minimum brightness if SSD1306 supports it, otherwise blank display). Buzzer OFF. LED OFF. Serial over USB still active. SD logging still active.
HIGH_ALERT: CAD-only rapid scan — skip the RSSI sweep entirely every cycle. Run one RSSI sweep every HIGH_ALERT_RSSI_INTERVAL_MS (default 10000 ms) as a sanity check. Immediate buzzer on any confirmed CAD tap. WiFi scanner still active. Maximum LED brightness. Fastest possible alert time.

Implementation requirements:

Add an OperatingMode enum to include/sentry_config.h: MODE_STANDARD, MODE_COVERT, MODE_HIGH_ALERT.
Add a global volatile OperatingMode currentMode = MODE_STANDARD in main.cpp. Protect reads/writes with stateMutex.
Add a HIGH_ALERT_RSSI_INTERVAL_MS constant to sentry_config.h (default 10000).
Button state machine in main.cpp Arduino loop() — replace or extend whatever button handling exists:

Single press (100–999 ms): cycle OLED screen (existing behavior — preserve it)
Double press within 1 second: toggle HIGH_ALERT (if already HIGH_ALERT, return to STANDARD)
Triple press within 2 seconds: toggle COVERT (if already COVERT, return to STANDARD)
Long press ≥ 3000 ms: mute buzzer (existing behavior — preserve it)


Gate each subsystem on the current mode:

loRaScanTask: if HIGH_ALERT, skip RSSI sweep unless millis() - lastHighAlertRssiMs >= HIGH_ALERT_RSSI_INTERVAL_MS
wifiScanTask: if COVERT, task should suspend itself via vTaskSuspend(NULL) after deinitializing WiFi. On mode change away from COVERT, reinitialize WiFi and resume via vTaskResume(hWiFiTask).
alertTask / buzzer: if COVERT, suppress all buzzer and LED output. If HIGH_ALERT, trigger buzzer immediately on any WARNING or CRITICAL event without the normal debounce delay.
displayTask: if COVERT, call display.clearDisplay(); display.display(); then skip all further rendering until mode changes.


Display current mode on the OLED — add a mode indicator to the existing dashboard screen header: [STD], [COV], [HI-ALT].
Log mode changes to serial and SD card with timestamp: [MODE] Switched to COVERT at 00:04:33
